## 作业 1：北极星指标

### 我的选择

**行程采纳率** —— 用户对话后是否有反馈表明采纳了行程。

### 为什么不是另外两个

| 指标 | 问题 |
|------|------|
| 生成行程数 | 鼓励刷量，不鼓励质量。一个人生成 10 份行程扔掉 9 份，数据漂亮但产品失败 |
| 用户满意度（点赞率） | 反馈率太低（100 人用完可能 5 个人点），样本不够做决策依据 |

### 面试话术

> "旅行规划器的核心价值不是生成一段文字，而是用户真的按照这个规划出发了并且体验好。所以我把北极星定为行程采纳率——通过对话结尾的主动询问和用户的复访行为来追踪。这个指标难追踪但离商业价值最近——一个采纳过的用户回来规划第二次旅行，比一百个生成完就走的人都有价值。"

### 追踪方案

| 信号 | 怎么捕获 | 置信度 |
|------|---------|:--:|
| 对话末尾用户明确说"好的/谢谢/就这个" | Dify 对话日志 | 中 |
| 用户 7 天内回来问"再帮我调一下这个行程" | 复访记录 | 高 |
| 用户主动反馈"去了，挺好的" | Dify 对话日志 | 最高 |
| 用户回来规划新目的地 | 复访记录 | 最高 |

## 作业 2：成本模型（真实数据）

> 数据来源：Dify 后台实测，单次简单对话

| 项目 | 数值 |
|------|------|
| 单次对话 Token | **1,655 tokens** |
| 速度 | 37.75 tokens/s |
| System Prompt 占比 | ~90%（约 1500 tokens） |
| 用户输入 + 模型输出 | ~10%（约 155 tokens） |

### DeepSeek V4 价格核算

```
输入：¥1/百万tokens    输出：¥2/百万tokens

单次成本 = 1550×0.000001 + 105×0.000002
         = 0.00155 + 0.00021
         = ¥0.00176
```

| 日活 | 人均轮次 | 日成本 | 月成本 |
|:--:|:--:|------|------|
| 100 | 3 | ¥0.53 | **¥16** |
| 1,000 | 3 | ¥5.28 | **¥158** |
| 10,000 | 3 | ¥52.80 | **¥1,584** |

### 降本方案：砍 System Prompt 冗余

| 动作 | 节省 | 是否伤体验 |
|------|:--:|:--:|
| Few-shot 完整示例砍成一行提示 | -200 tokens | 不伤——SOP 规则已覆盖 |
| 出行锦囊详细 list 缩成关键词 | -150 tokens | 不伤——模型自带常识 |
| 功能边界表合并成一句话 | -100 tokens | 不伤——"不付款不导航不竞价" |

→ System Prompt 1500 → **1050 tokens**，月成本省 **28%**

## 作业 3：用户漏斗

```
进入对话                    100 人
    │                       
    ▼                       
追问确认画像                 70 人      ← 🔴 最该优化
    │                       
    │  30 人流失原因：
    │  · 不知道该怎么回答追问
    │  · 追问语气像填表（TC-004 已修复）
    │  · 只是想随便看看，没想真规划
    │                       
    ▼                       
生成完整行程                 55 人      ← 🟡 其次
    │                       
    │  15 人流失原因：
    │  · 输出不符合预期
    │  · 画像识别错了（学生给了白领方案）
    │  · 目的地不在知识库范围 → 以后都靠通用知识兜底了
    │                       
    ▼                       
看完并给反馈                 20 人
    │                       
    │  35 人沉默消费
    │  · 行程末尾需要加引导："这个行程对你有帮助吗？👍/👎"
    │                       
    ▼                       
采纳了行程（北极星）           8 人
    │
    │  12 人虽然看完但没出发
    │  · 正常损耗——旅行是低频行为
    │                       
    ▼                       
复访（回来规划新行程）         3 人      ← 验证了北极星的价值
```

### 当前最该优化：第一层——画像确认

**理由**：70 人进来 30 人连画像都没确认就走了。这层转化从 70% → 85%，后面每一层的水量跟着涨。而且 TC-004 修复后这层的体验已经提升了——下一步该跟踪数据看改了有没有效果。

### 每层对应的 PM 动作

| 漏斗层 | 当前 | 目标 | 动作 |
|--------|:--:|:--:|------|
| 画像确认 | 70% | 85% | 已修 TC-004；观察一周数据 |
| 生成行程 | 79% | 85% | 补测 TC-002/TC-003 白领和老人 |
| 给出反馈 | 36% | 50% | 行程末加引导句 |
| 采纳行程 | 8% | — | 长期追踪，北极星不下短期目标 |
| 复访 | 3% | — | 季度看趋势 |
