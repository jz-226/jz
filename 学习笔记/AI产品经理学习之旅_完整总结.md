## 二、模块一：AI 基础认知

### 学到的核心概念

| 概念 | PM 级理解 | 类比 |
|------|---------|------|
| **Token** | 文本最小处理单位，也是计费单位 | 买菜按斤称 |
| **Embedding** | 把文字变成一串数字（向量），相似的东西数字也相似 | 人民币换美元——同一笔钱，更方便国际通用 |
| **Transformer** | 能同时理解整句话所有词关系的架构，GPT 的「T」 | 速读高手——一眼看整页，不是从左往右读 |
| **三者关系** | Token 是原料 → Embedding 是数字化 → Transformer 是处理引擎 | — |

| 概念 | PM 级理解 |
|------|---------|
| **Prompt 工程** | 不是写代码，是跟 AI 沟通清楚需求然后反复确认。三步流程：①大白话倒出来 → ②让 AI 反哺 Prompt → ③彩排 3 组案例定稿 |
| **RAG** | 检索增强生成。两阶段：建库（加载→切块→向量化）+ 问答（检索→生成）。PM 核心关注：幻觉防控、检索精度、成本 |
| **Agent vs Chatbot** | Chatbot 只聊天，问一句答一句。Agent 能干活——工具调用 + 多步推理 + 记忆 |

### 技能产出
- ✅ 会写结构化 Prompt（三步流程法）
- ✅ 能画出 RAG 五步全链路（建库 + 问答）
- ✅ 能区分什么时候用 Chatbot、什么时候必须用 Agent
- ✅ 能跟算法工程师无障碍沟通 Token/Embedding/Transformer

## 三、模块二：AI 旅行规划器（实战项目）

### 项目定位
面向学生/社会人士/老年人三大人群的 AI 旅行规划 Agent，覆盖穷游打卡/周末深度/慢游辅助三大场景。

### 项目全流程

| 阶段 | 做了什么 | PM 技能 |
|------|---------|---------|
| **产品需求定义** | 3 类用户画像 × 3 种场景 = 3×3 模式矩阵，定义 5 大核心功能和 3 条产品边界 | 用户分层、需求分析、功能边界定义 |
| **Prompt 系统设计** | 4 层 Prompt 架构（角色层/规则层/约束层/示例层），含 SOP 对话流程 + Few-shot 示例 | Prompt Engineering、对话设计 |
| **MVP 搭建** | Dify + DeepSeek V4，Bot 上线运行 | 零代码 AI 应用搭建 |
| **RAG 知识库** | 上传 5 份苏州旅行资料，配置 Top-K=5 + 相似度阈值 0.65 | 知识库构建、检索参数调优 |
| **评测与迭代** | Bad Case 分析（迪士尼错排 → 新增错峰算法），3 轮 Prompt 迭代 | 产品评测、数据驱动迭代 |
| **产品 PPT** | 6 页精华版，含技术架构 + 评测框架 + Bad Case + 个人收获 | 产品演示、面试展示 |
| **项目复盘** | 3 个关键决策回顾 + 如果重来的改进 | PM 反思能力 |

### 关键决策记录

| 决策 | 原因 | 学到什么 |
|------|------|---------|
| Coze → Dify 迁移 | Coze 积分限制迭代频率 | 工具选型看长期成本，不是短期爽感 |
| 用豆包生成 PPT | PM 的核心价值是内容决策，排版执行交给 AI | 多 Agent 协作是 AI Native 工作方式 |
| 边做边补基础 | Bug 的根因永远在基础上 | 基础概念不是学完就丢的 |
