# AI 基础概念（PM 级理解）

> 整理：2025.06.25 | 用途：面试时能跟算法工程师聊 15 分钟不露怯

---

## 一、Token

### 一句话定义
文本被切碎后的最小处理单位。

### 生活类比
Token 就像菜市场买菜的"斤"——你问 AI 问题，它按 Token 数（斤数）计费，不是按"句话"（棵数）计费。

### 举例
```
原文：「我今天想去苏州玩」

中文切 Token → 「我」「今天」「想」「去」「苏州」「玩」 （6 token）

英文 "I want to visit Suzhou"
→ "I" "want" "to" "visit" "Su" "zhou" （7 token，Suzhou 被拆成两个）
```

### PM 要关心
- 花的钱 = 输入 Token 数 + 输出 Token 数
- 中文比英文省 Token（中文集约）
- 模型有上下文窗口上限（比如 128K Token ≈ 一本《三体》的长度）

---

## 二、Embedding

### 一句话定义
把文字/图片/声音变成一串数字（向量）。

### 生活类比
Embedding 就是给每个词拍了一张"坐标地图"——意思相近的词，在地图上住得近。

```
"苏州" → [0.23, -0.71, 0.05, ..., 0.18] （比如 1024 个数字）

"苏州" 和 "南京" 的 embedding 距离 ≈ 很近
"苏州" 和 "西瓜" 的 embedding 距离 ≈ 很远
```

### PM 要关心
- RAG 的核心就靠它：问题 → Embedding → 查知识库 → 找距离最近的文档 → 喂给 LLM
- Embedding 质量决定检索质量——垃圾进垃圾出

---

## 三、Transformer

### 一句话定义
让模型能同时看到所有词之间关系的神经网络架构。

### 生活类比
以前的模型是**手电筒**——一次照亮一个词，从左读到右。
Transformer 是**探照灯**——一次照亮整句话，所有词之间的关系同时看清。

```
"小明把苹果给了小红，她很高兴"

手电筒（旧模型）：读到最后不知道"她"是谁
探照灯（Transformer）：同时看到"她"和"小红"的关系 → 知道高兴的是小红
```

### PM 要关心
- GPT 的「T」就是 Transformer
- 这是 ChatGPT 比老式对话机器人聪明那么多的根本原因
- 「注意力机制（Attention）」是 Transformer 的核心创新

---

## 四、三者关系（一句话）

```
Token → Embedding → Transformer

文本先被切成 Token（分词）
→ 每个 Token 转成 Embedding（数字化）
→ Transformer 处理这些数字，理解它们之间的关系（理解语义）
→ 最后输出结果
```

## 五、面试速记卡

| 被问 | 答 |
|------|-----|
| "Token 是什么" | "文本被切碎后的最小处理单位，也是计费单位。中文比英文省 Token。" |
| "Embedding 是什么" | "把文字变成一串数字，相似的东西数字也相似。RAG 检索就靠它。" |
| "Transformer 是什么" | "GPT 的 T。它让模型能同时理解一句话里所有词的关系，不是从左读到右。" |
| "三者什么关系" | "Token 是原料，Embedding 是数字化，Transformer 是处理引擎。" |
