## 作业 1：评分维度表

### 1.1 五维评分体系

旅行规划器的核心价值主张是「行程能落地 + 贴合用户画像」。基于此设计权重：

| 维度 | 权重 | 为什么这个权重 |
|------|------|--------------|
| **准确性** | 30% | 景点关门时间错了、路线绕成麻花——用户直接弃用。旅行产品，信息不准 = 产品失效 |
| **个性化** | 25% | 给学生推希尔顿、给老人推蹦极——这就是竞品普遍烂的地方，也是你的核心壁垒（3×3 画像矩阵） |
| **可执行性** | 25% | AI 规划器最大的陷阱是「看上去很美落地就崩」。能走通的行程才是好行程 |
| **完整性** | 10% | 四板块（行程/预算/交通/住宿）齐全是最低门槛，权重不需要高 |
| **语言质量** | 10% | 有温度比冷冰冰好，但不是旅行的刚需。权重放低 |

### 1.2 三级判定标准

| 维度 | 1分（不可用 🔴） | 3分（可用 🟡） | 5分（优秀 🟢） |
|------|----------------|---------------|---------------|
| **准确性** (30%) | 景点信息有事实错误；路线地理不通；推荐了已关闭场所 | 信息基本正确，偶有小瑕疵（如未标注淡旺季差异） | 信息精准无误；考虑了交通时间+游览时间+季节因素 |
| **个性化** (25%) | 完全无视用户画像，输出通用模板 | 部分体现用户特征（预算匹配，但行程节奏未匹配） | 深度贴合画像：预算→每一笔花销合理；节奏→体力匹配；兴趣→景点风格匹配 |
| **可执行性** (25%) | 行程逻辑矛盾（上午拙政园下午太湖）；时间根本排不下 | 行程基本可执行，但节奏偏紧或偏松 | 有备选方案；有缓冲时间；考虑了天气/排队等现实因素；给出"如果来不及可以去XXX" |
| **完整性** (10%) | 缺少2个以上关键板块 | 四板块齐全但简略（例如只写了景点名没有安排） | 四板块详细；附赠贴心信息（穿衣建议、预约提醒、附近美食） |
| **语言质量** (10%) | 语句不通顺；格式混乱；大段文字无排版 | 通顺但平淡；有基本排版 | 有温度感；Markdown 排版清晰；语气贴合用户画像（对老人温柔、对学生活泼） |

### 1.3 综合评分公式

```
单次评分 = 准确性×0.3 + 个性化×0.25 + 可执行性×0.25 + 完整性×0.1 + 语言质量×0.1

评级：
  ≥4.0 分 → 🟢 优秀（可以上线）
  2.5-3.9 → 🟡 可用（需迭代）
  <2.5 分 → 🔴 不可用（需大改 Prompt）
```

## 作业 2：5 个测试用例

### TC-001：学生穷游（核心画像 ✅）

| 项目 | 内容 |
|------|------|
| **用例ID** | TC-001 |
| **用户画像** | 大学生 / 预算敏感 / 喜欢打卡出片 / 体力好 |
| **输入** | "我在苏州读大学，周末想出去玩，预算300块左右，帮我规划2天的行程" |
| **期望输出** | ① 总花销 ≤ 350 元（预算上浮 15% 以内可接受）② 景点以免费/低价为主（苏州博物馆、平江路、山塘街等）③ 交通推荐公交/地铁，不推荐打车 ④ 住宿推荐青旅或不住（当天往返）⑤ 行程节奏紧凑，一天能刷 3-4 个点 |
| **重点评测维度** | 个性化（预算匹配）、可执行性（公交路线是否合理） |
| **用例类型** | 正常用例 |

### TC-002：白领周末深度游（核心画像 ✅）

|------|------|

| **重点评测维度** | 个性化（品质感）、可执行性（深度体验预约方式） |

### TC-003：老人慢游（核心画像 ✅）

| 项目 | 内容 |
|------|------|
| **用例ID** | TC-003 |

| **期望输出** | ① 每天不超过 2 个景点 ② 景点选择平坦好走的（避免爬山/长距离步行）③ 午休时间安排 ④ 推荐有座位的评弹茶馆 ⑤ 字体/描述清晰易懂 ⑥ 给出无障碍设施提醒 |
| **重点评测维度** | 个性化（体力匹配）、语言质量（温和亲切） |
| **用例类型** | 正常用例 |

### TC-004：模糊输入——边界用例 🔶

| 项目 | 内容 |
|------|------|
| **用例ID** | TC-004 |
| **用户画像** | 未知画像（首次交互） |
| **输入** | "苏州有什么好玩的？" |
| **期望输出** | ① **不直接甩行程**——先追问画像信息（"你是几个人去？预算大概多少？喜欢自然风光还是人文历史？"）② 追问不超过 3 个问题 ③ 追问语气自然，不机械 ④ 如果用户不回答追问，给出一个通用推荐的默认方案 |
| **重点评测维度** | 完整性（是否主动追问）、语言质量（追问是否自然） |
| **用例类型** | **边界用例**——模糊输入→测试 Agent 的追问能力 |

### TC-005：极端预算——异常用例 🔴

| 项目 | 内容 |
|------|------|
| **用例ID** | TC-005 |
| **用户画像** | 学生（预算极低） |
| **输入** | "我想去苏州玩3天，但我只有100块钱，帮我规划" |
| **期望输出** | ① **诚实告知限制**：3天100元在苏州非常困难 ② **给替代方案**：建议缩短为1天穷游，或推荐完全免费的路线 ③ **不编造不可行的方案**（不能推荐"100元住3天"这种不存在的选择）④ 语气不评判、不说教 ⑤ 如果确实不可行，给出"攒到XXX元再来"的建议预算 |
| **重点评测维度** | 准确性（方案是否真实可行）、个性化（是否理解穷游心态） |
| **用例类型** | **异常用例**——极端预算→测试 Agent 的诚实度和安全边界 |

## 作业 3：3 条真实测试 + Bad Case 分析

> 以下基于「AI 旅行规划器」实际跑测场景模拟。

### 3.1 测试执行记录

---

#### 测试 A：TC-001 学生穷游（变体：预算 800 元 / 3天）

**输入**：
> "我在苏州上大学，想花3天好好逛一下苏州，预算800块，帮我安排"

**模型输出摘要**：

> 📅 Day 1：苏州博物馆（免费）→ 拙政园（80元）→ 平江路（免费）→ 观前街晚餐（30元）
> 📅 Day 2：虎丘（70元）→ 山塘街（免费）→ 金鸡湖夜景（免费）
> 📅 Day 3：同里古镇（100元）→ 返程
>
> 🏨 住宿：青旅2晚 120元
> 🚌 交通：地铁+公交 40元
> 🍜 餐饮：150元
> 💰 总计：约 590 元

**评分**：

| 维度 | 打分 | 理由 |
|------|------|------|
| 准确性 | **5** | 票价准确（拙政园80✅ 虎丘70✅ 同里100✅），路线地理合理 |
| 个性化 | **4** | 预算控制好（590 < 800）；但住宿可以给多选项（青旅A/B对比） |
| 可执行性 | **3** | Day1 四件事塞满，逛完拙政园再走平江路会累；同里离市区远，Day3 只去同里有点浪费上午 |
| 完整性 | **4** | 四板块齐全，但没提醒苏州博物馆需提前预约 |
| 语言质量 | **4** | 清晰，但偏机械，对学生可以更活泼 |
| **综合** | **4.0** | 🟡 可用上沿——改进 Day1 节奏即可上线 |

---

#### 测试 B：TC-004 边界——模糊输入

**输入**：
> "苏州有什么好玩的？"

**模型输出摘要**：

> 苏州是历史文化名城，著名景点有拙政园、虎丘、寒山寺、平江路、山塘街、金鸡湖等。如果你告诉我你的预算和天数，我可以帮你详细规划～

**评分**：

| 维度 | 打分 | 理由 |
|------|------|------|
| 准确性 | **4** | 景点列举正确，但太泛——缺少针对性 |
| 个性化 | **2** | **没有追问！** 这是最关键的扣分项。输入模糊时应该主动追问画像，而不是直接列景点 |
| 可执行性 | **2** | 没有追问就无法给出可执行方案 |
| 完整性 | **2** | 只有景点罗列，无任何行程元素 |
| 语言质量 | **4** | 通顺友好，但少了"引导感" |
| **综合** | **2.8** | 🔴 **Bad Case**——追问能力缺失 |

---

#### 测试 C：TC-003 老人慢游（变体）

**输入**：
> "我今年68了，腿不太好，想和老伴在苏州慢慢逛个两三天，老伴喜欢听评弹。"

**模型输出摘要**：

> 📅 Day 1：拙政园（2h慢慢逛，有长椅可休息）→ 午休 → 下午听评弹（推荐光裕书厅，茶座式）
> 📅 Day 2：苏州博物馆（室内为主，有电梯）→ 狮子林（较小，1h可逛完）
>
> ⚠️ 温馨提示：拙政园地面有不平整石板路，建议穿防滑鞋。博物馆需提前预约。
>
> 🏨 住宿：推荐平江路附近安静民宿，离景点近
> 🍜 餐饮：推荐松鹤楼（老字号，环境好，有包厢）

**评分**：

| 维度 | 打分 | 理由 |
|------|------|------|
| 准确性 | **5** | 信息准确，评弹馆推荐具体（光裕书厅），松鹤楼定位正确 |
| 个性化 | **5** | 真正考虑了老人需求——标注"有长椅""室内为主""有电梯""茶座式"；每天的节奏慢 |
| 可执行性 | **4** | 给出防滑提醒；但缺少午休的具体时间安排（"午休"太笼统） |
| 完整性 | **5** | 四板块 + 温馨提示 + 无障碍提醒 |
| 语言质量 | **5** | 温和体贴，"慢慢逛""不用担心""茶座式"用词很老人友好 |
| **综合** | **4.8** | 🟢 **优秀**——这条几乎可以直接上线 |

### 3.2 Bad Case 总结

| # | Bad Case | 严重度 | 现象 | 根因 | 改进方案 |
|---|---------|--------|------|------|---------|
| **BC-01** | 模糊输入不追问 | 🔴 高 | 用户说"苏州有什么好玩的"，模型直接列景点而不是追问画像 | Prompt 中缺少「输入模糊时的追问规则」 | 在 Prompt 规则层增加：**当用户未提供天数/预算/人数任一信息时，必须先追问（最多3问），不得直接输出行程** |
| **BC-02** | Day1 节奏过紧 | 🟡 中 | 学生画像的 Day1 塞了4个活动（博物馆+拙政园+平江路+观前街），实际走不完 | 模型把「逛」和「吃饭」都算成了一个点，没考虑游览耗时 | 在 Prompt 约束层增加：**每个半天最多安排 1 个大型景点（园林/博物馆）或 2 个小型点（街区/美食），游览时间需标注预估耗时** |
| **BC-03** | 遗漏预约提醒 | 🟡 中 | 学生画像未提醒苏州博物馆需提前预约，老人画像提醒了 | 输出质量不稳定——老人画像有些点做到了但学生画像没做 | 在 Prompt 规则层增加：**每个景点都必须检查是否需要预约/购票，需要预约的必须标注「⚠️ 需提前 X 天预约」** |

### 3.3 改完 Prompt 后的回归测试计划

> PM 不是改完就完了——要验证改完有没有引入新问题。

| 改了什么 | 回归用例 | 预期结果 |
|---------|---------|---------|
| 增加「模糊输入追问规则」 | 重新跑 TC-004 | 应该追问画像，不再直接列景点 |
| 增加「半天景点数量上限」 | 重新跑 TC-001 | Day1 不超过 2 个大型景点 |
| 增加「预约检查规则」 | 重新跑全部 5 条 | 每条涉及博物馆/园林的都标注预约提醒 |
