## 模块 4：核心流程

### 主流程

```
用户打开 Bot
    │
    ▼
Bot 发送欢迎语，引导用户输入旅行需求
    │
    ▼
用户输入 ──→ 目的地 / 天数 / 预算 / 兴趣标签 / 注意事项
    │
    ▼
Bot 生成行程 ──→ 按模板输出（景点 → 交通 → 餐饮 → 预算估算）
    │
    ▼
用户查看 → 结束
```

### 异常流程

| # | 异常场景 | Bot 行为 |
|---|---------|---------|
| 1 | 用户缺少关键信息（目的地/天数/预算未提供） | Bot 友好追问，一次只问一个缺失项 |
| 2 | 用户预算过低（如 50 元/天） | 提示预算偏紧，给出建议金额，不直接拒绝 |
| 3 | 网络超时 / API 无响应 | 保留已输入信息不清空，提示用户稍后重试 |
| 4 | 用户指定日期的景点关闭（闭馆/限流） | 自动检索开放状态，遇冲突时替换同类景点或提示用户 |
| 5 | Bot 生成内容不合规（幻觉景点/违规内容） | 触发重新生成，连续失败返回通用友好提示 |
| 6 | 用户输入偏离旅行话题（闲聊、问无关问题） | 友善引导回正轨，不冷处理、不生硬——「我目前主要专注在旅行规划上~ 你想去哪里玩？我可以帮你做一份行程！」 |

## 模块 5：功能详情——行程生成

### 入口
用户完成信息输入后，Bot 开始生成。

### 输入参数

| 参数 | 必填 | 格式 | 示例 |
|------|------|------|------|
| 目的地 | 是 | 城市（+区域可选） | 苏州 / 苏州·姑苏区 |
| 天数 | 是 | 整数 1-7 | 2 |
| 预算 | 是 | 整数（元/天） | 300 |
| 兴趣标签 | 否 | 自由输入 | 园林 / 美食 / 拍照 |
| 注意事项 | 否 | 自由文本 | 「带老人，少走路」|

### 生成逻辑

1. 从 RAG 知识库检索匹配「目的地 + 兴趣标签」的景点
2. 按预算过滤（门票累加 ≤ 日均预算 × 天数）
3. 按天数分配景点，每天 ≤ 3 个，考虑距离顺路
4. 如有注意事项（如「带老人」），优先过滤掉不适合的景点
5. 每天输出：上午景点 → 午饭 → 下午景点 → 晚饭 → 交通
6. 计算总预算

### 输出格式

```
📋 你的 [目的地] [天数] 日行程

【Day 1 — 预算约 ¥XXX】
🌅 上午：[景点]（门票 ¥XX）
  → 建议游览 X 小时
🍜 午饭：[餐厅]（人均 ¥XX）
🌤 下午：[景点]（门票 ¥XX）
🍲 晚饭：[餐厅]（人均 ¥XX）
🚇 交通：[交通说明]

💰 总预算：¥XXX
📝 提示：[闭馆提醒/天气建议等]
```

## 模块 3（补）：功能范围——V1.0 边界

✅ **做**：
- 单次行程生成（输入需求 → 输出完整日行程）
- 苏州地区旅行规划

❌ **不做**：
- 多轮修改行程
- 酒店/机票预订
- 其他城市
- 语音输入
