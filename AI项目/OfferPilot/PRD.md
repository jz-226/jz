# OfferPilot · PRD

## 一、产品概述

### 1.1 产品名
OfferPilot（AI Offer Operating System）

### 1.2 一句话
一个帮求职者从目标设定走到拿到 Offer 的 AI 操作系统。

### 1.3 核心问题
ChatGPT 能回答问题，能生成学习计划。但它不会追踪你执行了没、不会根据你的进展调整计划、不会在你卡住的时候主动找你。OfferPilot 解决的就是这个"从回答到执行"的断层。

## 二、目标用户

### 第一阶段：校招生
- 2027届本科/硕士应届生
- 目标岗位：互联网、AI、产品、设计、开发
- 痛点：不知道从哪开始、不知道自己做对了没、没有人持续追踪进度

### 后续扩展
- 转行求职者
- 职场 1-3 年想跳槽的人

## 三、核心功能

### 3.1 目标管理
- 用户设定目标（岗位/城市/时间）
- 系统围绕目标运行所有功能

### 3.2 每日追踪
- 每天主动问"今天做了什么离目标更近的事"
- 记录回答，追踪连续打卡

### 3.3 每周报告
- 每周日自动生成周报
- 对比上周，判断离目标近还是远

### 3.4 能力画像
- 记录用户的技能变化
- 跟目标 JD 做差距分析

### 3.5 面试记录
- 每次面试后记录复盘
- 下次面试前回顾历史

## 四、MVP 范围

第一版只做：
- 目标设定
- 每日一问 + 记录
- 每周总结
- 飞书表格持久化

不做：
- JD 自动分析
- Offer Score 量化
- 多 Agent
- 手机 App

## 五、用户流程

```
用户打开 Bot
  → Bot："你的目标是什么？"
  → 用户回答
  → Bot："你现在什么阶段？每天能投入多少时间？"
  → 用户回答
  → Bot："收到。从今天起我每天问你一件事。"

每天：
  → Bot："今天做了什么离目标更近的事？"
  → 用户回答
  → Bot 记录到飞书表格
  → Bot："这件事为什么重要？"
  → 用户自己说出来

每周日：
  → Bot 自动生成周报
```

## 六、成功指标

| 指标 | 目标 |
|------|:--:|
| 第一周留存 | > 60% |
| 第二周留存 | > 40% |
| 用户主动说"本周总结"的频次 | 每周 ≥ 1 次 |
| 用户秋招结束后的反馈 | "这个产品帮了我" |
