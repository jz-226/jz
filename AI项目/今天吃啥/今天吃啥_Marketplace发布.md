# 今天吃啥 · Dify Marketplace 发布

## 📌 应用名称

**今天吃啥（AI 吃饭管家）** — Agent 版

---

## 📝 一句话简介

记住你的口味和预算，帮你决定今天吃什么。绝不让你连吃两天一样的菜。真正的 Agent——有持久记忆，有数据库。

---

## 📖 应用概述

"今天吃啥"是一个有记忆的 AI 吃饭管家。跟普通 Chatbot 不同——它不只是聊天，它通过飞书多维表格做持久化记忆存储。

每次推荐前，Agent 自动查飞书表格——你最近三天吃了什么？有什么忌口？预算偏好？然后基于这些数据生成个性化推荐。推荐完自动写回表格，下次自动避开重复食材。

你能随时打开飞书表格，看到自己的完整饮食记录——花了多少钱、吃了什么、每天营养搭配。

---

## ✨ 核心功能

- 🧠 **持久化记忆**：飞书表格存储，关了窗口不丢失
- 🚫 **自动去重**：最近三天吃过的食材自动排除
- 💰 **预算控制**：推荐总价严格不超预算，超了给替代方案
- 🥬 **营养搭配**：按中国居民膳食指南，每顿至少一个蔬菜
- 👥 **多用户隔离**：不同用户打开 Bot，数据自动隔离
- 🎲 **随机推荐**：不知道吃什么？Agent 帮你定，但不乱定

## 🛠️ 设置步骤

### 前置条件（5 分钟）

你需要两样东西：

| 工具 | 用途 | 注册地址 |
|------|------|------|
| **Dify 账号** | 运行 Workflow | https://dify.ai |
| **飞书账号** | 数据库存储 | https://www.feishu.cn |

---

### 第一步：创建飞书多维表格（3 分钟）

1. 打开 **https://fvs5liujf2.feishu.cn/base/** → 登录飞书
2. 右上角 **"+ 新建"** → **"多维表格"** → 名字填 **"吃饭记录"**
3. 加四列：

| 列名 | 类型 |
|------|------|
| 吃了什么 | 多行文本 |
| 主料 | 文本 |
| 花了多少钱 | 数字 |
| 用户 | 文本 |

4. 复制地址栏链接 → 提取 `app_token` 和 `table_id`

示例：`https://xxx.feishu.cn/base/ABC123?table=tblXYZ789`

```
app_token = ABC123
table_id  = tblXYZ789
```

---

### 第二步：创建飞书应用（3 分钟）

1. 打开 **https://open.feishu.cn/app** → 扫码登录
2. 右上角 **"创建企业自建应用"** → 名字填 **"今天吃啥 Agent"**
3. 左侧 **"权限管理"** → 搜索 `bitable` → 勾选 ✅ **bitable:app**
4. 左侧 **"版本管理与发布"** → **"创建版本"** → 填 `v1.0` → **"发布"**
5. 左侧 **"凭证与基础信息"** → 复制 **App ID** 和 **App Secret**

---

### 第三步：给应用授权表格（1 分钟）

1. 打开你的飞书表格
2. 右上角 **"…"（更多）** → **"权限设置"** → **"关联应用"**
3. 搜索 **"今天吃啥 Agent"** → 添加 → 权限给 **"可编辑"**

---

### 第四步：导入 Dify Workflow

1. 在 Dify Marketplace 找到本模板 → 点 **"导入"**
2. 打开 Workflow → 修改以下节点：

| 节点 | 改什么 |
|------|------|
| **节点 2（HTTP 拿 Token）** | Body 里：`app_id` 改成你的、`app_secret` 改成你的 |
| **节点 3（HTTP 查历史）** | URL 里：`Ysreb14G6aarTSsNVxkcaCVYnwd` 改成你的 app_token、`tbloSgrPT93gaMpt` 改成你的 table_id |
| **节点 5（HTTP 写入）** | URL 和 Body 同上 |

3. 右上角 **"发布"**

---

### 第五步：测试

打开 Bot → 输入：

> "中午吃啥，自己做，20块，1人"

期望收到一份推荐（菜名 + 价格 + 营养说明 + 已存档提示）。

打开飞书表格刷新 → 看到一行新记录 → 🎉 完成！

## 📊 技术架构

- **Agent 引擎**：Dify Workflow + DeepSeek V4
- **数据库**：飞书多维表格（通过 Open API）
- **记忆机制**：HTTP 读写飞书 → 每次推荐前查历史 → 推荐后写回
- **用户隔离**：Dify `sys.user_id` 传入飞书 → 按用户过滤数据
