# AI 今天吃啥 · Dify 搭建指南

## 第一步：创建 Bot

1. 打开 Dify → **创建应用** → 选 **Chatbot**
2. 应用名：**今天吃啥**
3. 模型：**DeepSeek V4**
4. System Prompt：粘贴 Prompt 文件里"复制线以下"的全部内容
5. 点 **发布**

---

## 第二步：配置对话开场白

Dify 应用设置 → **对话开场白** → 填：

```
你好！我是你的吃饭管家 🍳

第一次用的话，告诉我：
- 几个人吃？
- 自己做还是点外卖？
- 预算多少？

比如："一个人，自己做，20块"
```

---

## 第三步：开启对话变量（记忆）

左边 → **变量** → **对话变量** → 添加三个：

| 变量名 | 类型 | 默认值 | 用途 |
|------|------|------|------|
| `user_profile` | String | `{}` | 存忌口/偏好/常用模式 |
| `recent_meals` | String | `[]` | 存最近3天的主料 |
| `weekly_nutrition` | String | `{}` | 存本周吃了什么类型 |

> ⚠️ 如果 Dify 版本没有对话变量，可以跳过——对话历史本身就够用。

---

## 第四步：测试

### 测试 1：新用户首次使用

**输入**："中午吃啥"

**期望**：追问模式/人数/预算 ≤ 3个问题

---

### 测试 2：正常推荐

**输入**："省钱自己做，1人，20块"

**期望**：
- 1荤1素
- 每道菜有价格
- 总价 ≤ 20元
- 有营养说明
- 末尾有 "💾已存档"

---

### 测试 3：去重验证

**输入**："明天继续，还是省钱自己做，20块"

**期望**：
- 跟测试2的主料**不能重复**
- 末尾标注避开的主料

---

### 测试 4：预算冲突

**输入**："我想吃红烧排骨，20块自己做"

**期望**：
- 诚实告知排骨贵
- 给替代方案（红烧鸡块）
- 不硬推超预算方案

---

### 测试 5：偏好记忆

**输入**："老样子"

**期望**：
- 自动调取上次的模式和预算
- 不再追问基础信息

---

### 测试 6：口味偏好

**输入**："我不吃香菜，爱吃辣"

然后下一顿再说"中午吃啥"。

**期望**：
- 永远不推荐含香菜的菜
- 优先推荐辣味做法

---

## 第五步：面试讲法

> "我做了四个 Agent——旅行规划、儿童康复、学情本、吃饭管家。吃饭管家是我唯一一个有记忆系统的 Agent。它记住你的忌口、口味、最近三天吃了什么、本周营养摄入——然后基于这些数据帮你决定今天吃什么。这比旅行规划器进了一步——旅行规划器每次对话是独立的，吃饭管家需要跨对话记住状态。"

---

## 你要提供的素材

| 素材 | 用途 |
|------|------|
| 常见菜市场菜价（鸡胸肉/猪肉/青菜/鸡蛋） | Prompt 里菜价估算的基础 |
| 本地应季蔬菜清单 | 让推荐更接地气 |

先不提供也行——Prompt 里已经写了"约XX元"的估算机制。
