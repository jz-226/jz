## 📋 节点速查表

| 节点号 | 类型 | 干什么 |
|:--:|------|------|
| 1 | 开始 | — |
| 2 | HTTP 请求 | 拿飞书 API 钥匙 |
| 3 | LLM | 判断用户是想"推荐"还是"查历史" |
| 4 | 条件分支 | 分两条路：推荐 / 查历史 |
| 5 | HTTP 请求 | 查飞书表格——最近吃了什么 |
| 6 | LLM | 生成今日推荐（避重复食材） |
| 7 | HTTP 请求 | 把推荐结果写回飞书表格 |
| 8 | HTTP 请求 | 查飞书表格——历史记录 |
| 9 | LLM | 汇总历史 → 给建议 |
| 10 | 结束 | 输出结果 |

---

## 🔗 节点之间的线怎么连

```
1 → 2 → 3 → 4 ──recommend──→ 5 → 6 → 7 → 10
              │
              └──history────→ 8 → 9 ─────→ 10
```

---

## ⚡ 三个关键配置

### 飞书 Token（节点 2）
```
URL: https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal
Body: {"app_id":"你的App_ID","app_secret":"你的App_Secret"}
```

### 查历史（节点 5 + 8）
```
URL: https://open.feishu.cn/open-apis/bitable/v1/apps/O2CvwV7H6i6f0Nkj7Dhc6rsGncZ/tables/tbl4fTtUZZGMCnXb/records?page_size=20
Header: Authorization: Bearer {{feishu_token}}
```

### 写入记录（节点 7）
```
URL: https://open.feishu.cn/open-apis/bitable/v1/apps/O2CvwV7H6i6f0Nkj7Dhc6rsGncZ/tables/tbl4fTtUZZGMCnXb/records
Header: Authorization: Bearer {{feishu_token}}
Body: {"fields": {"吃了什么": "{{节点6的输出}}"}}
```
